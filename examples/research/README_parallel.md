# Parallel Research Agent System

A hierarchical research system implementing the **SMS Deep Researcher Agent Interaction Flow** that uses file-based context management to coordinate specialized sub-agents conducting parallel, adaptive information gathering through iterative search processes.

## Architecture Overview

### Core Components

```
User Query → Lead Agent Analysis → Research Context Creation
    (`docs/research/research-session.md`)
                ↓
    Parallel Sub-Agent Deployment
    ├── Technical Researcher ────── docs/research/technical-findings.md
    ├── Market Researcher ──────── docs/research/market-findings.md
    ├── Academic Researcher ────── docs/research/academic-findings.md
    ├── Regulatory Researcher ──── docs/research/regulatory-findings.md
    └── Social Impact Researcher ── docs/research/social-findings.md
                ↓
    Lead Agent reads all research files → Final Synthesis
```

### Lead Agent (Research Coordinator)

**Role**: Query analysis, strategy development, and result synthesis

**Responsibilities**:
- Creates and maintains research context files
- Develops comprehensive research strategy
- Spawns specialized sub-agents for parallel investigation
- Synthesizes findings from research reports into final response

### Sub-Agents (Intelligent Research Specialists)

**Role**: Domain-specific adaptive research and intelligent filtering

**Available Specialists**:
- **Technical Researcher**: Implementation details, architecture, technical feasibility
- **Market Researcher**: Market trends, business models, competitive landscape
- **Academic Researcher**: Scientific studies, theoretical foundations, peer-reviewed sources
- **Regulatory Researcher**: Legal frameworks, compliance, policy implications
- **Social Impact Researcher**: Social implications, cultural factors, user behavior

**Responsibilities**:
- Analyze requirements within their research dimension
- Perform iterative search cycles with dynamic query refinement
- Act as intelligent filters to identify relevant information
- Create detailed research reports and findings
- **Critical Focus**: Research and analysis only, no final synthesis

## Key Features

### File-Based Context Management

- **Shared Research Context**: `docs/research/research-session.md`
  - Original user query and objectives
  - Research strategy and dimension assignments
  - Progress tracking across all sub-agents
  - Key discoveries and cross-agent insights

- **Specialized Research Reports**: Auto-generated by sub-agents
  - Comprehensive findings for specific research dimensions
  - Source quality assessments and relevance scores
  - Iterative search progression and strategy adaptations

### Parallel Research Strategy

- **Simultaneous Investigation**: Multiple sub-agents explore different dimensions concurrently
- **Intelligent Filtering**: Sub-agents evaluate source quality and information relevance
- **Adaptive Search**: Dynamic strategy adjustment based on real-time findings
- **Cross-Pollination**: Agents reference other agents' findings through shared file system

## Usage

### Basic Usage

```python
from parallel_research_agent import parallel_research_agent

# Initialize research query
query = "Analyze the impact of blockchain technology on supply chain management"

# Create initial state
state = {
    "messages": [{"role": "user", "content": query}],
    "files": {}
}

# Execute parallel research
result = parallel_research_agent.invoke(state)

# Access final report
final_report = result["messages"][-1].content
research_files = result.get("files", {})
```

## Research Process Flow

### 1. Research Context Initialization
- User submits research query
- Lead agent creates research context file (`docs/research/research-session.md`)
- Establishes shared knowledge base and research objectives

### 2. Research Strategy Development
- Lead agent analyzes query complexity and scope
- Develops multi-faceted research plan with different investigation angles
- Determines optimal sub-agent specializations needed

### 3. Parallel Sub-Agent Deployment
- Lead agent assigns research dimensions to appropriate sub-agents
- Provides research context file reference and specific investigation focus
- Sub-agents deployed **simultaneously** for parallel research

### 4. Sub-Agent Research Cycle
- Sub-agent reads shared research context file
- **Multi-Step Adaptive Search Process**:
  - Initial broad search using specialized tools/databases
  - Intelligent filtering to identify high-value sources
  - Iterative search refinement based on findings
  - Dynamic query adaptation and strategy adjustment
- Creates detailed research report
- Updates shared research context file with key discoveries

### 5. Result Synthesis
- Lead agent reads all research reports from file system
- Synthesizes findings across all research dimensions
- Updates research context file with final analysis
- Formulates comprehensive response to user query

## Benefits

- **Coordinated Parallel Research**: File-based context prevents duplicate efforts
- **Comprehensive Coverage**: Shared context ensures no research gaps
- **High-Quality Filtering**: Specialized agents provide domain expertise
- **Adaptive Intelligence**: Dynamic strategy adjustment improves research effectiveness
- **Scalable Architecture**: Easy to add new specialized research dimensions
- **Context Preservation**: File system prevents information loss during complex research

## Comparison with Sequential Research

| Aspect | Sequential Research | Parallel Research |
|--------|-------------------|-------------------|
| **Speed** | Linear time complexity | Parallel execution |
| **Depth** | Single perspective | Multi-dimensional analysis |
| **Coverage** | May miss aspects | Comprehensive coverage |
| **Coordination** | Limited context sharing | File-based coordination |
| **Scalability** | Doesn't scale well | Easily add new dimensions |

## Troubleshooting

### Common Issues

1. **Missing API Key**: Ensure `TAVILY_API_KEY` is set
2. **Rate Limits**: Adjust search frequency in sub-agent prompts
3. **File Conflicts**: Ensure unique file paths for each research dimension
4. **Memory Usage**: Monitor file system usage during large research tasks

### Performance Optimization

- Adjust `max_results` in search parameters
- Customize recursion limits based on research complexity
- Implement caching for frequently accessed sources
- Use content filtering to reduce context size
